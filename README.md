## 命令
* nmap -p port ip 查看服务器端口是否开放
* iptables -L -t mangle 查看DSCP标签配置
* arp -h 查看arp信息
* 格式化输出printf代替echo
* tr -d可以快速删除字符，也可以tr [options] source-char-list replace-char-list
* -n检查语法不执行，-x跟踪脚本执行详情，也可以在脚本中使用set -x打开跟踪开关 ，set +x关闭
单个+号表示执行的语句，两个+号表示子shell中执行，shell的函数都在子shell中执行。
* if grep  pattern file > /dev/null可以代替-s -q选项，测试其他命令退出状态也可以使用此方法
* stty -echo; read pass < /dev/tty; stty echo可以关闭回显并读取密码
* cat > new_file; echo > new_file; touch new_file都可以创建新文件
* 内存 - free，-m以M为单位显示
* 磁盘 - iotop -oP，pidstat -d 1; sar -b 1 10
* cron - /var/spool/cron/中可查看任务配置，/var/spool/mail/可查看任务执行日志

## 其他
> 针对不同业务场景制定可用性目标：无人车场景需要远超99.999%（年故障时间5分钟），云业务99.999%足够

> 每个设备独立线程，内存消耗4M，1000个设备4G，考虑内存消耗；秒级采集，对CPU来说是long time，不用考虑CPU压力  
SQL计算网络密集型和CPU密集型，线程数量设置为CPU数量或者CPU*2或3  
流式计算，网络密集型；doris，CPU密集

> 结构化数据与非结构化数据差异，系统间交互数据尽量结构化，除非特殊场景考虑性能提升  
proto文件省略key字段内容传输，client和server端都进行存储结构，序列化与反序列化，压缩与解压

> 使用IP最后网段进行哈希，方便调试。

> 系统设计时考虑依赖服务的可用性，以及其不可用时对系统的影响，考虑是否需要对依赖服务的监控、预警以及容灾。
